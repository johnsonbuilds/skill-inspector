Refactor Skill Inspector from an Asset-centric architecture to a Package-aware architecture.

Important:

Do NOT remove existing functionality.

Do NOT rewrite the project.

Extend the current architecture.

Tasks:

1. Add new models:

* Category
* SkillPackage
* PackageAsset

2. Add HermesAdapter.discover_packages()

Hermes organizes knowledge as Skill Packages:

The `skills/` **directory structure is as follows:**

skills/
├── category-name/          ← Category folder
│   ├── DESCRIPTION.md      ← Description of the category (optional)
│   └── skill-name/         ← Single skill folder
│       ├── SKILL.md        ← Required
│       ├── references/     ← Optional
│       ├── templates/      ← Optional
│       ├── scripts/        ← Optional
│       └── assets/         ← Optional


Note there is an exception: `skills/dogfood/` has no `DESCRIPTION.md` and no sub-skill folders; it is itself a single skill (containing `SKILL.md` + `references/` + `templates/`). Similarly, `skills/yuanbao/` and `skills/software-development/` also place skills directly rather than categories. Therefore, not all categories strictly follow the `DESCRIPTION.md` pattern—some skills are placed directly under the `skills/` root directory.

3. Create PackageClassifier.

Only classify SKILL.md.

Do not classify references independently.

4. Create PackageDuplicateDetector.

Compare only SKILL.md content.

Ignore references/templates/scripts/assets.

5. Create PackageReportGenerator.

Report:

* Total Categories

* Total Skill Packages

* Total Assets

* Category Breakdown

* Package Type Distribution

* Largest Packages

* Most Complex Packages

* Duplicate Skill Packages

Keep existing Asset-based implementation intact.

This is an additive refactor, not a rewrite.
