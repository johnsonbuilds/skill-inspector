# Skill Inspector v0.4.2

## Theme

Executive Report Refinement

v0.4.1 successfully introduced:

* Runtime Governance
* Skill Utilization Analysis
* Unused High-Cost Skills
* Governance Summary

The remaining issue is report readability.

Governance Recommendations currently dominate the report and bury the most important insights.

v0.4.2 focuses entirely on signal-to-noise ratio.

No new analysis engines.

No new scoring systems.

No new data sources.

Only report refinement.

---

# Goal 1

## Reduce Governance Recommendations Noise

Current issue:

The report contains 20+ recommendations.

Many are technically correct but not decision-critical.

Examples:

```text
Add category description
Convert to executable skill
Modularize scripts
```

These dilute the value of the report.

---

### New Rule

Only display:

```text
Critical
High
```

recommendations by default.

Hide:

```text
Medium
Low
```

from the main report.

---

### Optional Footer

At the end of Governance Recommendations:

```text
Additional Recommendations Hidden

Medium: 5
Low: 8
```

This preserves information without overwhelming readers.

---

# Goal 2

## Add Cleanup Priority Section

Current issue:

The same packages appear in:

* Highest Risk Packages
* Unused High-Cost Skills

This overlap is extremely valuable but currently invisible.

---

### New Section

Placement:

Immediately after:

```text
Unused High-Cost Skills
```

and before:

```text
Governance Recommendations
```

---

Section:

```text
## Cleanup Priorities
```

---

Logic

A package receives priority points from:

```text
Unused Skill
+
High Complexity
+
Low Health Score
+
High Risk Count
```

---

Display:

| Rank | Package | Priority Score | Reason |
| ---- | ------- | -------------: | ------ |

Example:

```text
1. productivity/powerpoint
   Unused + Complexity 135 + Risk

2. creative/popular-web-designs
   Unused + Template Bloat + Risk
```

Purpose:

Answer:

```text
If I only clean up 3 packages,
which 3 should I start with?
```

# Goal 3

## Improve Governance Summary

Current:

```text
• Health score below recommended level
• Utilization low
...
```

Good, but still generic.

---

### New Rule

Prioritize the strongest findings.

Maximum:

```text
5 bullets
```

ordered by impact.

---

Example:

```text
• Only 5.7% of installed skills have ever been used.
• 66 skills have never been used.
• productivity/powerpoint is the largest unused package.
• creative/popular-web-designs has the highest governance risk.
• Duplicate package risk remains low.
```

This makes the summary feel specific.

---

# Goal 4

## Improve Runtime Governance

Most Used Skills： only show top 5.
Recently Used Skills： only show latest 3 days.
Never Used Skills： only show top 10 by complexity. Omit the rest, just report the count
Category Utilization ：collapse all with 0 utilization and just report the count without listing them.

## Improve Unused High-Cost Skills Count 

Unused High-Cost Skills: should count it as "used" if view_count > 0. We should only consider it "used" when use_count > 0.



# Goal 5

## Clarify Most Valuable Packages

Current issue:

Value Score is a black box.

Example:

```text
baoyu-infographic
Value Score: 129
```

Users cannot understand why.

---

### Add Explanation Column

Display:

| Package | Health | Value Score | Drivers |
| ------- | -----: | ----------: | ------- |

Example:

```text
creative/baoyu-infographic

Health: 75

Value: 129

Drivers:
45 references,
high asset density,
good health
```

The score becomes interpretable.
