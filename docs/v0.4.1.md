# Skill Inspector v0.4.1

## Theme

Governance-First Report Layout

v0.4 successfully introduced runtime usage analysis using:

```text
~/.hermes/skills/.usage.json
```

However, the current report prioritizes inventory details over governance insights.

v0.4.1 reorganizes the report so users immediately see:

1. Overall health
2. Actual utilization
3. Key governance risks
4. Recommended actions

before any detailed inventory breakdown.

No major architecture changes.

---

# Goals

1. Improve report information hierarchy
2. Surface actionable governance insights earlier
3. Reduce report noise
4. Add runtime governance enhancements
5. Preserve all existing analysis capabilities

---

# New Report Order

Replace the current report ordering with:

```text
Summary

Skill Utilization Summary

Library Health Score

Governance Summary

Runtime Governance

Governance Recommendations

Highest Risk Packages

Unused High-Cost Skills

Most Valuable Packages

--------------------------------

Package Type Distribution

Asset Distribution

Category Breakdown

Duplicate Skill Packages
```

This ordering is mandatory.

The first screen of the report should communicate:

* How healthy the library is
* How much of it is actually used
* What the biggest risks are
* What actions should be taken

without requiring users to scroll through inventory tables.

---

# Goal 1

## Skill Utilization Summary

Placement:

Immediately after Summary.

Only show summary metrics.

Do NOT include tables.

Example:

```text
Installed Skills: 70

Used Skills: 4

Unused Skills: 66

Utilization Rate: 5.7%
```

Purpose:

Provide a quick understanding of library adoption.

---

# Goal 2

## Governance Summary

Placement:

Immediately after Library Health Score.

Purpose:

Provide a short executive summary.

Generate 3–5 bullet points.

Example:

```text
• Health score is below recommended level.
• Only 5.7% of installed skills have been used.
• 66 skills have never been used.
• Several large unused packages represent maintenance overhead.
• Duplicate package risk remains low.
```

This section should be concise and human-readable.

---

# Goal 3

## Runtime Governance Section

Placement:

Immediately after Governance Summary.

This section contains detailed utilization analytics.

Move all runtime details here.

---

### Most Used Skills

Display:

| Skill | Uses | Views | Last Used |
| ----- | ---: | ----: | --------- |

Data source:

```text
use_count
view_count
last_used_at
```

Sort:

```text
use_count DESC
```

---

### Recently Used Skills

Display:

| Skill | Last Used |
| ----- | --------- |

Sort:

```text
last_used_at DESC
```

---

### Never Used Skills

Display:

| Skill | Category |
| ----- | -------- |

Limit display:

```text
Top 20 entries
```

Show total count.

Example:

```text
66 skills have never been used.
```

---

### Category Utilization

Display:

| Category | Utilization | Used / Total |
| -------- | ----------: | ------------ |

Sort:

```text
utilization DESC
```

Do NOT generate recommendations from this table.

The table itself is sufficient.

---

# Goal 4

## Governance Recommendations Cleanup

Current behavior:

Every category with 0% utilization generates a recommendation.

Example:

```text
Category apple has 0% utilization
Category github has 0% utilization
Category media has 0% utilization
...
```

Remove this behavior completely.

Do not generate category-specific utilization recommendations.

---

Only generate library-level recommendations.

Examples:

```text
Only 5.7% of installed skills are actively used.

66 skills have never been used.

Consider archiving or removing unused skills.

Review large unused packages before adding new skills.
```

---

# Goal 5

## Unused High-Cost Skills

New section.

Placement:

Immediately after:

```text
Highest Risk Packages
```

and before:

```text
Most Valuable Packages
```

Purpose:

Identify unused skills that create the highest maintenance burden.

---

Only include:

```text
used = false
```

Sort by:

```text
complexity_score DESC
```

Display:

| Skill | Category | Complexity | References | Templates | Scripts |
| ----- | -------- | ---------: | ---------: | --------: | ------: |

Example:

```text
creative/popular-web-designs

productivity/powerpoint

research/research-paper-writing
```

This section should answer:

```text
If I clean up only a few skills,
which ones should I start with?
```

---

# Goal 6

Keep existing logic.

Placement:

After:

Unused High-Cost Skills

Show users which skills provide the highest value.
